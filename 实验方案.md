## 第一步：补全 `model.py`（唯一需要写代码的文件）

文件路径：`model.py`

### 1.1 补全 `CausalSelfAttention.forward`（第 35 行）

这是多头因果自注意力的核心，**禁止**使用 `nn.MultiheadAttention` / `F.scaled_dot_product_attention` 等封装，必须手搓。

实现步骤：

```python
def forward(self, x: torch.Tensor) -> torch.Tensor:
    B, T, C = x.size()

    # 1. 线性投影
    q = self.w_q(x)  # (B, T, C)
    k = self.w_k(x)
    v = self.w_v(x)

    # 2. 拆分多头：(B, T, C) → (B, n_head, T, d_head)
    q = q.view(B, T, self.n_head, self.d_head).transpose(1, 2)
    k = k.view(B, T, self.n_head, self.d_head).transpose(1, 2)
    v = v.view(B, T, self.n_head, self.d_head).transpose(1, 2)

    # 3. 缩放点积注意力
    scale = self.d_head ** -0.5
    att = (q @ k.transpose(-2, -1)) * scale  # (B, n_head, T, T)

    # 4. 因果掩码：位置 i 不能看到 j > i 的位置
    mask = torch.triu(torch.ones(T, T, device=x.device, dtype=torch.bool), diagonal=1)
    att = att.masked_fill(mask, float('-inf'))

    # 5. softmax + dropout
    att = torch.softmax(att, dim=-1)
    att = self.dropout(att)

    # 6. 与 V 相乘，合并多头，输出线性层
    y = att @ v                              # (B, n_head, T, d_head)
    y = y.transpose(1, 2).contiguous().view(B, T, C)  # (B, T, C)
    return self.w_o(y)
```

完成后**删除** `raise NotImplementedError(...)` 这一行。

**④ 三个简答题**（用自己的话，每题 2–4 句）

**Q1：因果掩码的作用是什么？**
> 提示：在自注意力中，位置 i 在训练时不应该"看到"它之后的字（否则模型可以直接抄答案）。因果掩码把注意力分数矩阵中 j > i 的位置填成 -∞，经 softmax 后这些位置的权重变为 0，确保每个位置只能基于自己和左侧历史生成输出，实现自回归行为。

**Q2：缩放点积为什么要除以 √d_k？**
> 提示：当 d_k 很大时，Q 与 K 的点积绝对值也随之增大，导致 softmax 的梯度趋于极小（饱和区），训练变慢甚至不稳定。除以 √d_k 让点积的方差保持在 1 附近，softmax 输出更平滑，梯度流动更健康。

**Q3：FFN 在 Transformer 块里承担什么职能？**
> 提示：注意力机制负责在序列不同位置之间交换信息，FFN 则在每个位置独立地对特征做非线性变换和升维-降维处理，增加模型的表达能力。两者分工协作：注意力做"跨位置通信"，FFN 做"逐位置特征处理"。

## 注意事项

- `model.py` 补全后，**3 处 `raise NotImplementedError` 全部要删掉**，不能只删一处。
- FFN 的 `nn.Linear` 等层必须在 `__init__` 中注册，不能在 `forward` 里临时 `new`（否则参数不被优化器管理）。
- 注意力掩码要在 softmax **之前** 应用（填 `-inf`，不是 0）。
- 超参对比时每跑完一组务必改名保存 checkpoint，避免覆盖。
