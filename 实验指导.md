# 实验指导：手搓因果注意力、前馈子层与古诗续写（字符级 GPT）

## 一、实验在做什么？

你可能听说过：像 GPT 这样的大模型，底层做的其实是一件事——**根据已经出现的文字，预测下一个字（Next Token Prediction）**。把「预测 → 接到句尾 → 再预测」循环下去，就有了续写、对话和作诗。

本实验的**必做部分**是在已搭好的**小型解码器式 Transformer** 上，在 `model.py` 中补全两处核心子层：

1. **多头因果自注意力**：**缩放点积**与**下三角（因果）掩码**；  
2. **位置前馈网络（FeedForward）**：**d_model → d_ff → GELU → d_model**，并配合 **Dropout**（与 Pre-LN、残差的衔接已在 `TransformerBlock` 中写好，你不必改块级结构）。

**你会练到的内容：** 多头自注意力、**\(\frac{QK^\top}{\sqrt{d_k}}\)**、**Causal Mask**、FFN 在块内的作用、自回归 **Temperature** 采样、**Loss 曲线**阅读。


**数据**来自 Hugging Face 的 **`Lifan-Z/Chinese-poetries-txt`**。`prepare_data.py` 会抽样约 **10万首**诗；**8G 显存**较舒适，**纯 CPU** 能跑、时间更长。

### 4.1 `CausalSelfAttention.forward`

按类内 docstring 完成：Q/K/V 与多头 reshape、\((QK^\top)/\sqrt{d_\text{head}}\)、**上三角掩码** + `masked_fill`（softmax **前**）、softmax、dropout、`att @ V`、合并头、`w_o`。

### 4.2 `FeedForward`

- **`__init__`**：根据 `d_model`、`d_ff`、`dropout` 注册所需层（例如两个 `nn.Linear`、`GELU`、`nn.Dropout`；也可用 `nn.Sequential` 等，**参数须登记在 `Module` 上**以便训练更新）。  
- **`forward`**：对 `x`（B, T, d_model）做前馈，**返回同形状**。不要改变 `TransformerBlock` 里「先 LN 再子层再残差」的调用方式。

**两处都补全并删除所有 `NotImplementedError` 后**，应能正常跑通 `python train.py` 与 `python predict.py`（需有效 checkpoint）。

## 七、常见问题

**Q：连不上 HuggingFace？** 换网络/镜像，或向教学方索取与脚本一致的数据包。

**Q：显存/内存不足？** 在 `train.py` 顶栏改小 `BATCH_SIZE`，再改 `D_MODEL`、`N_LAYER`、`BLOCK_SIZE` 等。

**Q：注意力部分 NaN？** 检查掩码与 softmax 维，避免某行对 key 全为 `-inf`。

**Q：FFN 补全后 shape 报错？** 确认最后一维是 `d_model`，且**仅在 `__init__` 中**创建 `nn.Linear` 等，**不要**在 `forward` 里每步 `nn.Linear` 临时 new（会丢参数、无法训练）。

**Q：只补了一处、另一处还 raise？** 两处都补全后训练才能正常前向+反向（否则会在对应处中断）。
